# Manual Testing Project: Webook App

## Project Overview
This project involves a comprehensive **Manual Testing** process for the **Webook** app. The goal was to ensure the quality, usability, and functional integrity of the booking flow and user account management.

## Testing Scope
**Functional Testing:** Validating login, event searching, and booking processes.
**UI/UX Testing:** Checking for layout consistency and user flow.
**Negative Testing:** Testing with invalid data to ensure proper error handling.

## Deliverables
**Test Cases:** Detailed Excel sheet covering 30+ test scenarios.

## Test Plan & Environment

### 📱 Device Specifications
To ensure the application's quality in a real-world usage scenario, the testing was conducted on a physical mobile device:
- **Device:** [HUAWEI MatePad]
- **OS:** [HarmonyOS 2.0.0]
- **Connection:** Tested across various network conditions (Wi-Fi, 4G, and Offline mode).

### 🛠 Testing Methodology & Techniques
The testing followed a structured approach to cover all critical user paths on the mobile platform:

- **Functional Testing:** Verified core features including user authentication, event search functionality, and filtering mechanisms.
- **Negative Testing:** Simulated edge cases such as network disconnection during operations and rapid multiple interactions to ensure system stability.
- **Interrupt Testing:** Observed app behavior when switching between background/foreground and handling connectivity loss.
- **Mobile UI/UX Assessment:** Evaluated the responsiveness of touch targets, text alignment, and layout consistency across different screens.
- **Performance Observation:** Monitored app load times and button responsiveness to ensure a smooth user experience.

### 📊 Results Summary
- **Total Test Cases:** 30+ Scenarios
- **Execution Status:** 100% Pass
- **Critical Defects:** 0
